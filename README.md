<h3>프로젝트를 하면서 느낀점</h3>
보통 it교육을 들으면 java에서 포기한다는 얘기를 많이 들었던 것 같습니다. 그래서 이파트에선 공부를 많이 할수밖에는 없었고
조원들도 의욕이 너무 과다해서인지, 너무 많은 기능을 집어넣은데다 ppt분량도 너무 많아서 발표시간에만 한시간 반정도의 시간이 걸렸던것 같습니다.
객체의 개념에 대해 이해하는데 시간이 오래걸렸고, 또 그걸 메서드에 사용하는데도 애를 많이 먹었던것 같습니다. 아무리 디버깅을 해봐도 도대체 어디서 오류가 터지는건지
알수가 없어서 메서드 외적인 부분에서 오류를 찾기시작했는데, db에서 오류가 생겼다는걸 알게되었습니다. 그래서 저혼자 db의 오류를 수정하고 다시 내용을 업데이트하게 되었습니다.
조장이나 목소리가 큰조원은 바쁜데 이런게 왜중요하냐고 오히려 이상한 사람 취급했기에, 화가 많이났지만 일단 참고 넘어가게 되었습니다.
다들 의욕이 너무 앞섰던지 ppt분량만 45페이지 가까이되어서 올릴수있는 용량을 초과하였기에, 발표 ppt 자료에는 제가 넣은 기능만 올려두었습니다.
